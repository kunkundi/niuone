# 策略研究说明

简体中文 | [English](README_EN.md)

本文档集中说明 NiuOne 中用于研究实验的信号筛选、模拟决策和复盘机制。相关能力仅运行在本地研究与模拟环境中，不连接券商账户，不执行真实资金交易，也不构成投资建议。

## 1. 研究流程

NiuOne 将研究流程拆分为四层：

1. **候选生成**：根据当前启用的规则扫描公开行情数据，形成待观察样本。
2. **信息补充**：按需聚合指数、板块、资金、市场热度和近期消息。
3. **模拟判断**：结合规则约束、账户状态与可选模型输出，生成模拟记录。
4. **归档复盘**：保存输入摘要、来源状态、模拟操作和结果，便于后续检查。

策略输出是实验性信号，不应作为真实交易依据。行情源或模型源暂时不可用时，系统会记录来源状态，并按当前可用信息继续或跳过对应流程。

## 2. 独立策略

设置页直接选择一套当前策略，一次只激活一套。基础策略、Z 哥、李大霄、板块潮汐和预设文字策略是同级、互斥的独立策略；每套策略独立提供候选范围、评分、买入、卖出、仓位和模型 Prompt。未启用策略不会进入本轮新开仓扫描或决策上下文。

切换策略不会篡改历史持仓归因。已有持仓继续按买入时记录的 `strategy_mark` 执行原策略退出纪律。预设文字策略使用基础扫描结果作为原始候选池，再按文字规则独立完成模拟判断；如果文字为空，则不会产生新的模拟建仓记录，只对已有模拟持仓执行风险检查。

### 2.1 用户：如何启用与触发

1. 在设置页把“当前独立策略”设为“板块潮汐”，对应配置为 `DASHBOARD_ACTIVE_STRATEGY=sector_tide`。该设置可热应用，保存后从下一轮扫描开始生效，不需要重启 Dashboard。
2. 新候选和模型买卖复用实战页面的 B1 扫描计划，由 Dashboard 进程内置调度器按 `DASHBOARD_B1_SCHEDULE_TIMES` 触发；板块潮汐没有另一套独立选股定时器。
3. 如需立即运行，可在实战页面点击“手动触发选股及买卖策略”。一次完整执行包含全市场扫描、候选生成、模型判断和执行层风险复核。
4. 09:25 的扫描只使用竞价结果形成候选，不会在 09:25–09:30 静默期模拟成交；可执行动作会等待 09:30 后重新检查价格、交易时段和风险限制。

调度职责分为两个进程：

| 工作 | 进程 | 主要配置 | 行为 |
|---|---|---|---|
| 选股和模型买卖 | Dashboard | `DASHBOARD_B1_SCHEDULE_ENABLED`、`DASHBOARD_B1_SCHEDULE_TIMES` | 到点或手动触发后运行扫描，并把当前激活策略的候选送入模拟决策 |
| 本地自动退出 | Cron Scheduler | `DASHBOARD_B3_EXIT_TIME`、`DASHBOARD_TIME_EXIT_TIME` | 在配置时间刷新持仓行情并检查结构止损、退潮、时间窗、2R 和 2ATR 等规则 |

自动退出是离散时间检查，不是券商条件单或逐笔行情实时监控。页面刷新只读取状态，不会触发自动成交。切换到其他策略后，板块潮汐不再产生新候选，但已有板块潮汐持仓仍按保存的策略标记接受退出检查。

## 3. 策略套件

| 策略组 | 包含的代理信号 | 研究定位 |
|---|---|---|
| 基础策略 | 突破确认、趋势回踩 | 通用技术形态观察 |
| Z 哥 | 少妇 B1、B2 确认、B3 中继、超级 B1、退出风控 | 趋势与节奏类规则实验 |
| 李大霄 | 低估蓝筹、底部发育、逆向情绪、去杠杆防守 | 价值与防守类规则实验 |
| 板块潮汐 | 主线领航、轮动初升、冰点修复 | 市场状态、行业轮动与板块内相对强度 |

### 3.1 基础策略

- **突破确认**：平台或前高突破后回踩站稳，作为趋势确认样本。
- **趋势回踩**：强趋势标的回踩 BBI / EMA 不破，作为低位观察样本。

### 3.2 Z 哥规则组

该规则组参考 [`zettaranc-skill`](https://github.com/lululu811/zettaranc-skill) 中整理的公开方法，当前包含：

- **少妇 B1**：关注 J 值低位、N 型上移、缩量回调和 BBI 约束。
- **B2 确认**：观察 B1 后的放量阳线确认，并过滤明显滞后或偏离 BBI 过远的样本。
- **B3 中继**：观察 B2 后的小阳线或十字星，以及分歧转一致特征。
- **超级 B1**：观察放量破位后的缩量企稳，以及 J 值仍处低位的形态。

退出与风险规则包括：前低或入场 K 线约束、固定风险阈值、防卖飞评分、分批退出、S1 / S2 / S3 逃顶、出货形态、白线 / BBI 破位、峰值回撤、ATR 吊灯保护，以及 B2、B3、超级 B1 的时间约束。

### 3.3 李大霄规则组

该规则组参考 [`li-daxiao-skill`](https://github.com/sherjy/li-daxiao-skill) 的政策、价值、底部发育、逆向情绪和杠杆风控框架，并使用高流动性蓝筹、低位企稳、低换手、缩量低波动、反追高和风险标的过滤作为可执行代理信号。

### 3.4 板块潮汐

板块潮汐每轮先对同一批高流动性股票生成一个横截面快照，再按“市场状态 → 行业潮位 → 板块内个股强度”筛选，行业是硬门控而不是个股分数的加分项。

- **主线领航**：进攻或轮动行情中的领先行业，个股须处行业前 20%，只做突破或 EMA20 附近缩量回踩；8% 是单票绝对上限，实际仓位由动态风险预算决定。
- **轮动初升**：行业排名和广度快速改善，个股须处行业前 30%，拒绝单日涨幅超过 7% 或距 EMA20 超过 1.5 ATR 的追高；6% 是单票绝对上限。
- **冰点修复**：仅在防守解除后的修复状态做率先转强行业，T+2 未确认退出；4% 是单票绝对上限，实际仓位由修复状态风险预算决定。

风险预算随市场状态切换。进攻、轮动、修复的单笔权益风险预算分别为 0.30%、0.20%、0.10%，策略内组合未实现止损风险上限为 1.50%、0.80%、0.30%，单行业风险上限为 0.60%、0.40%、0.20%；对应总仓上限为 45%、30%、15%，行业敞口上限为 12%、10%、6%，防守状态全部归零并禁止新仓。同一行业最多 2 只。有效损失距离按“结构止损距离 + max(近 60 日向下跳空 P95，0.5 ATR 占比) + 0.20% 费用滑点”计算，实际单票上限取风险预算反推仓位与注册绝对上限 8%/6%/4% 的较小值。行业分数连续两个交易日低于 55、市场复合风险硬停止且行业转弱、或策略时间窗未兑现时退出；盈利达到 2R 先减半，余仓使用 2 ATR 跟踪。行业资金流缺失时使用量能参与度作为明确标记的后备指标，不会把缺失值当作净流入。

#### 用户：仓位和退出如何执行

- 模型必须为每条 BUY/SELL 给出 100 股整数倍数量。执行层不会替模型补默认仓位，也不会把超额订单自动缩小后成交。
- 候选必须有有效结构止损，结构距离须同时不超过 6% 和 1.5 ATR；缺少历史跳空/ATR 缓冲时禁止买入。
- 实际买入数量会依次接受单票动态上限、市场总仓、同行业数量、行业敞口、单笔风险、策略组合风险和策略内行业风险检查，任何一项超限都整笔拒绝，并在决策日志中记录原因。
- 市场状态预算是板块潮汐的最高许可值；全局盘面指引、`DASHBOARD_MAX_TOTAL_POSITION_PCT` 或现金缓冲更严格时，执行层取其中最小值，不会因全局配置更宽松而放大板块潮汐预算。
- 总仓和行业敞口按整个模拟账户计算；同行业数量会计入账户中所有带同一行业标签的持仓。组合止损风险和行业止损风险只汇总板块潮汐策略持仓。
- 主线领航 5 个交易日未兑现、轮动初升 3 个交易日不延续、冰点修复 T+2 未确认，均在尾盘时间检查中退出。
- `2R目标 = 平均成本 + 2 ×（平均成本 - 入场结构止损）`。首次达到 2R 减半，余仓在跟踪线高于成本后按 `持仓最高价 - 2 × ATR` 退出。

#### 开发者：状态、评分和数据契约

市场状态由同一轮扫描快照生成。市场综合分使用核心指数趋势 25%、涨跌家数广度 25%、中位涨跌幅 15%、涨跌停结构 15%、样本 20 日趋势 10% 和量能参与度 10%。状态机口径如下：

| 状态 | 判定 |
|---|---|
| `offensive` | 市场分数 ≥65 且上涨广度 ≥55%，并通过状态确认 |
| `rotation` | 未触发防守且不满足进攻条件，并通过状态确认 |
| `recovery` | 上一确认状态为防守，本轮原始状态已经解除防守 |
| `defensive` | 复合硬停止，或市场分数 <40；风险预算归零 |

复合硬停止要求指数破位、市场广度破位和跌停扩散同时成立。除硬停止外，状态变化通常需要连续两轮出现同一原始状态；首次没有历史状态时直接采用本轮结果。

行业评分权重及潮位阈值：

| 因子 | 权重 |
|---|---:|
| 20 日相对强度分位 | 25% |
| 5 日相对强度分位 | 15% |
| 排名加速度分位 | 15% |
| EMA20 以上广度 | 20% |
| 20 日新高占比 | 10% |
| 行业资金流；缺失时使用量能参与度 | 10% |
| 行业成交额流动性分位 | 5% |

行业至少需要 3 个有效成员，个股至少需要 55 根日 K。`leading` 要求行业分数 ≥75 且 20 日相对强度分位 ≥70；`improving` 要求分数 ≥65、排名加速度 ≥15 且 5 日相对强度分位 ≥65；分数 <45 或 20 日相对强度分位 <35 为 `lagging`，其余为 `weakening`。

主线领航、轮动初升、冰点修复的入选评分阈值分别为 8.0、8.2、8.5；板块内强度分位分别至少为 80、70、70。扫描结果必须把 `market_regime`、行业潮位、结构止损、跳空缓冲、有效损失距离和各级风险预算随候选写入缓存，执行层会用实时成交价和共享风险参数重新计算，不信任模型自行给出的风险数值。

核心实现位于 `app/strategies/scoring/sector_tide.py`、`app/strategies/sector_tide_risk.py` 和 `app/trading/practice_trader.py`。专项回归入口为 `python3 -m unittest -v tests.test_sector_tide_strategy`，全量验证入口为 `./scripts/validate.sh`。

上述名称仅用于标记项目中的规则实验，不代表相关原作者参与、认可或背书本项目。二次分发相关说明时，请保留对 `zettaranc-skill` 与 `li-daxiao-skill` 的引用。

## 4. 模拟决策情报包

模拟流程可以将多个来源压缩为一个结构化上下文，并随决策日志保存。默认可能包含：

- 盘面监控指引和隔夜市场摘要；
- A 股指数、A50、美股指数或期货、黄金和原油等行情；
- 板块涨跌、行业资金、成交热度与换手信息；
- 候选样本的近期消息，以及行业与市场数据的共振或背离；
- 模拟账户现金、总仓位、持仓比例、盈亏状态和规则标记。

每次实战选股会复用扫描器在当前选股范围内已经取得的实时报价，按当时的上涨/下跌家数和涨跌停广度重算盘面标签；数据覆盖率不足、快照过期或仍处于 9:25 竞价阶段时，回退到最近的竞价、午盘或盘后报告。即使本轮没有候选股，也会刷新并记录盘面上下文。

仓位比例按 `价格 × 数量 ÷ 当前模拟总权益` 计算。日志会记录单次变化占模拟总权益的比例，以及变化后的单项和总仓位比例。

相关配置：

| 配置项 | 说明 |
|---|---|
| `DASHBOARD_DECISION_INTELLIGENCE_ENABLED` | 是否启用结构化情报包 |
| `DASHBOARD_DECISION_INTELLIGENCE_TTL_SECONDS` | 聚合数据缓存时间 |
| `DASHBOARD_DECISION_INTELLIGENCE_MAX_ITEMS` | 各类信息的最大条目数 |
| `DASHBOARD_TRADE_DISCIPLINE_TEXT` | 自定义模拟纪律文本 |
| `DASHBOARD_MAX_OPEN_POSITIONS` | 最大模拟持仓数量参考 |
| `DASHBOARD_MAX_NEW_BUYS_PER_DECISION` | 单轮新增模拟样本数量参考 |
| `DASHBOARD_MAX_SINGLE_POSITION_PCT` | 单项模拟仓位比例参考 |
| `DASHBOARD_MAX_TOTAL_POSITION_PCT` | 总模拟仓位比例参考 |
| `DASHBOARD_MIN_CASH_RESERVE_PCT` | 模拟现金保留比例参考 |

百分比配置默认主要用于模型上下文和研究纪律；Z 哥和板块潮汐等带注册硬限制的策略，会在模拟执行层把相关全局总仓/现金限制与策略上限取更严格者执行。无论是否硬拦截，这些设置都不应被视为真实交易账户的风险保障。

## 5. 配置方式

优先在 dashboard 设置页维护当前独立策略、文字规则和模拟纪律。每轮只启用一套策略；基础策略、Z哥、李大霄、板块潮汐和后续新增策略的候选、买入、卖出、仓位与 Prompt 规则互不混用。对应环境变量包括：

| 配置项 | 说明 |
|---|---|
| `DASHBOARD_ACTIVE_STRATEGY` | 当前独立策略：`base`、`zettaranc`、`li_daxiao_bottom`、`sector_tide` 或 `preset_text` |
| `DASHBOARD_PRESET_STRATEGY_TEXT` | 自定义预设文字规则 |
| `DASHBOARD_STOCK_UNIVERSE` | 选股范围，多选值为 `st`、`chi_next`、`star_market`、`main_board`；默认仅主板 |
| `DASHBOARD_TRADE_DISCIPLINE_TEXT` | 额外模拟纪律 |

旧版 `DASHBOARD_STRATEGY_SOURCE` 与 `DASHBOARD_ENABLED_PERSONA_STRATEGIES` 仅用于无缝读取历史配置；新设置保存后以 `DASHBOARD_ACTIVE_STRATEGY` 为准。

本地配置默认保存在 `.local-data/dashboard.env`。该文件可能同时包含模型密钥和管理凭据，不应提交到 Git 或复制到公开上下文。

## 6. 扩展独立策略

策略代码统一放在 `app/strategies/`：

- `registry.py`：策略元数据、分组、别名、启停和设置项；
- `scoring/`：指标、硬过滤、各策略评分器及多策略比较引擎；
- `selection.py`、`policy.py`：候选准入、展示筛选与仓位策略；
- `attribution.py`、`performance.py`：策略标记、归因和绩效汇总；
- `exits.py`、`prompts.py`：策略专属退出规则和模型提示词片段。

`app/screening/multi_strategy.py` 负责行情获取、全市场扫描和缓存编排，`app/trading/practice_trader.py` 负责账户、风控和模拟成交编排。旧模块名适配器集中在 `app/compat/`，新代码应从 `strategies` 包导入。

新增内置策略时：

1. 在 `app/strategies/registry.py` 增加 `label`、`color`、`desc`、`scorer`、`profile`、`position_limit_pct` 和 `aliases`；
2. 在 `app/strategies/scoring/` 的对应文件中实现 `score_xxx(rows)`，并在 `scoring/__init__.py` 的显式映射中注册；
3. 如有专属退出或决策语义，只修改 `exits.py` 或 `prompts.py`，无需把规则写回扫描器或交易器；
4. 为评分边界、空数据和异常行情补充自动化测试；
5. 运行 `./scripts/validate.sh` 完成验证。

扫描器会遍历当前启用策略的 scorer，并将 `strategy_meta` 输出给 dashboard 与模拟复盘模块。

## 7. 使用边界

- 不要将模型生成内容视为事实，应回查原始数据和信息来源。
- 不要使用本项目替代持牌机构服务、专业风险评估或个人独立判断。
- 历史回放、规则评分和模拟结果不代表未来表现。
- 扩展数据源时，应确认数据授权、调用频率、隐私和再分发要求。
